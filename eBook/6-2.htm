    1. 传统的Windows打印程序设计

在本实验中，我们通过研究示例程序来了解传统Windows的打印过程。

在Windows下，基本的打印过程有如下六步：

1) 调用CreateDC() 函数获得打印机DC (设备描述环境) 。

2) 调用StartDoc() 函数启动文档。

3) 调用StartPage() 函数打印一页。

4) 把文档提交给打印机DC。

5) 调用EndPage() 函数结束一页。

对文档中要打印的每一页都重复步骤 3) 至 5) 。

6) 调用EndDoc() 函数结束打印任务。

为了确定与系统相连接的打印机的型号，应用程序必须搜索用户的WIN.INI文件或调用Windows的EnumPrinters() API函数。而调用EnumPrinters() 函数是最容易的方法。

EnumPrinters() 函数的定义如下所示：

      BOOL EnumPrinters(

          DWORD Flags,

          LPTSTR Name,

          DWORD Level,

          LPBYTE pPrinterEnum,

          DWORD cbBuf,

          LPDWORD pcbNeeded,

          LPDWORD pcReturned,

      ) ;

函数各参数按顺序描述如下：

Flags

表明所需打印机类型的标志

Name

打印机对象的名称

Level

打印机信息结构的类型

pPrintereEum

指向打印机信息结构的指针

cbBuf

打印机信息数组的大小

pcbNeeded

指向保存复制到打印机信息数组中的字节数变量的指针

pcReturned

指向保存复制到数组中的打印机信息结构的变量的指针

其中某些参数根据打印机的不同描述而具有许多不同的值。但是，大部分情况下，关心的只是获得缺省的打印机。为了对应用程序进行编程，以便要求Windows提供缺省的打印机，可使用如下代码：

      PRINTER_INFO_5 printerInfo5[3] ;

      DWORD needed, returned;

      EnumPrinters(PRINTER_ENUM_DEFAULT, NULL, 5, (LPBYTE) printerInfo5,

     sizeof(printerInfo5) , &needed, &returned) ;

如果上述代码段执行成功，那么在第一个打印机信息结构的pPrinteName成员中就有了缺省打印机的名称。可以按以下方式来访问该成员：

      printerInfo5[0].pPrinterName

为了给打印机创建DC，需要准确地知道打印机名称。为了获得该DC，可调用CreateDC() ：

      HDC printDC;

      printDC = CreateDC(NULL, printerInfo5[0].pPrinterName, NULL, NULL) ;

这里，把打印机的系统名称作为第二个参数，其余参数都应该为NULL。

一旦有了用户打印机的DC，就可以开始打印文档。首先，调用Windows API函数StartDoc() ，开始把文档发送到系统的后台打印程序中。函数调用成功则返回打印机任务的标识符；否则返回小于或等于零的值。

StartDoc() 的一个参数是指向DOCINFO结构的指针。必须初始化此结构的成员才能调用StartDoc() 。在大多数情况下，只需将此结构的大小放在cbSize成员中，把指向文档名称的指针放在lpszDocName成员中，其余成员可以是NULL或零。整个过程类似于如下形式：

      char docName[] = “RectangleDoc” ;

      DOCINFO docInfo;

      docInfo.cbSize = sizeof(docInfo) ;

      docInfo.lpszDocName = docName;

      docInfo.lpszOutput = NULL;

      docInfo.lpszDatatype = NULL;

      docInfo.fwType = 0;

 

      result = StartDoc(printDC, &docInfo) ;

      if (result <=0 )

      {

          MessageBox(0, “StartDoc() failed” ,

              “Basic Print App”, MB_OK | MB_ICONERROR) ;

          return;

      }

如果调用StartDoc() 成功，则可以通过调用StartPage() 来打印第一页。StartPage() 函数只有单一参数，即打印机的DC。函数调用成功则返回大于零的值，否则返回小于或等于零的值。打印第一页的代码类似如下程序片断：

      result = StartPage(printDC) ;

      if (result <= 0 )

      {

          MessageBox(0, “StartPage() failed”,

              “Basic Print App” , MB_OK | MB_ICONERROR

          return;

      }

一旦开始打印第一页，那么只需要把输出导向打印机DC。由于屏幕上的点的大小与大多数打印机上的点的大小不同，所以通常还需要缩放发往打印机的数据。

打印一页之后，可调用EndPage() 函数以结束当前页。与StartPage() 一样，EndPage() 需要打印机DC作为其惟一的参数。并且，EndPage() 调用成功，则返回大于零的值；如果不成功，返回零或小于零的值。结束一页的代码类似于如下片断：

      result = EndPage(printDC) ;

      if (result <= 0 )

      {

          MessageBox(0, “EndPage() = failed” ,

              “Basic Print App” , MB_OK | MB_ICONERROR

          return;

      }

至此，应用程序可再一次调用StartPage() 以打印另一页，也可调用EndDoc() 以结束打印任务：

      EndDoc(printDC) ;

    2. 在MFC应用程序中打印文本

进一步地，如果使用MFC AppWizard来组装应用程序，可以十分容易地获得基本的打印和打印预览功能。

为了获得这种功能，要确保在AppWizard的“Step 4 of 6”对话框中选择“Printing and print preview (打印和打印预览) ”选项，使AppWizard应用程序可获得打印机的名称和获取打印机DC的所有细节。读者只需要改进所生成的源代码就能够为应用程序产生适当的打印输出。

使用MFC的缺省打印功能时，应用程序的文档都在OnDraw() 函数中屏幕显示和发送到打印机。所不同的是，在屏幕显示的情况下，发送到OnDraw() 的DC对象是客户窗口；在打印的情况下，DC用于当前打印机。DC切换的灵活运用使某一函数既能向屏幕输出，也能向打印机输出，使产生WYSIWYG (What You See Is What You Get，所见即所得) 更加容易。

除了以正确的尺寸缩放打印机输出外，还可能要处理页码、页眉和页脚、字体以及Windows应用程序为打印具有专业外观的文档而必须完成的其他任务。MFC不仅通过创建打印机描述环境，而且通过在CView类中提供在各阶段都能够使视图窗口访问打印过程的各种成员函数，从而进行大量的打印文档的作业。

当编写带有打印功能的MFC程序时，通常需要为指定的应用程序改进缺省的MFC打印功能。CView类 (可从中派生自己的应用程序视图窗口) 定义了五个用于打印的MFC成员函数：OnPreparePrinting() 、OnBeginPrinting() 、OnPrepareDC() 、OnPrint() 和OnEndPrinting() ，能够用来对打印过程的不同阶段实施控制。

(1) OnPreparePrinting() 函数

必须在视图窗口类中重载这个函数并调用DoPreparePrinting()函数。DoPreparePrinting() 函数显示Print对话框，并创建打印机DC。在OnPreparePrinting()函数中可以改变Print对话框中显示的一些值。

当使用MFC AppWizard来创建自己的应用程序，并为此应用程序选择Printing and print preview选项时，AppWizard自动重载了OnPreparePrinting() 函数，如下所示：

      BOOL CPrintAppView :: OnPreparePrinting(CPrintInfo * pInfo)

      {

          // 缺省的准备过程

          return DoPreparePrinting(pInfo) ;

      }

OnPreparePrinting() 重载了DoPreparePrinting() 函数，后者用来显示Print对话框。通过初始化作为惟一参数传递给该函数的CPrintInfo对象成员，可以控制在此对话框中显示的选项。例如，如果已知文档有三页，就可以按如下方式对CPrintInfo对象的SetMinPage() 和SetMaxPageo成员函数加以调用：

      BOOL CPrintAppView :: OnPreparePrinting(CPrintInfo * pInfo)

      {

          // 缺省的准备过程

          pInfo -> SetMinPage(1) ;

          pInfo -> SetMaxPage(3) ;

          return DoPreparePrinting(pInfo) ;

      }

SetMinPage() 和SetMaxPage() 函数不仅告诉MFC要打印多少页，也给出了Print对话框的From和To框中显示的值。如果不调用 SetMinPage() 和SetMaxPage() ，那么From框缺省为1，To框为空。

使用OnPreparePrinting() 还没有解决该应用程序访问打印机DC的权限的问题，这就限制了确定当前文档和打印机的信息量。为此，可以修改OnBeginPrinting() 函数。

(2) OnBeginPrinting() 函数

MFC在调用OnPreparePrinting() 之后，调用OnBeginPrinting() 函数。此函数不做任何事情，但如果在应用程序的视图窗口类中重载了OnBeginPrinting() 函数，就可以创建打印任务所需要的GDI资源，包括画笔、画刷以及字体等。而且，因为OnBeginPrinting() 是视图窗口类能够访问打印机DC和CPrintInfo结构的第一个地方，当这些值依赖于打印机DC的设置时，可以在此处设置页数以及其他值。

当使用MFC AppWizard创建应用程序并为应用程序选择Printing and print preview选项时，AppWizard自动重载了OnBeginPrinting() 函数。

通过调用诸如GetTextMetrics() 的打印机DC成员函数，就可以确定用于打印机的正确的字符高度，还可以计算出每页的行数以及需要显示整个文档的页数。下面就是OnBeginPrinting() 函数用来计算字符尺寸和页数的示例：

void CPrintAppView :: OnBeginPrinting(CDC * pDC, CPrintInfo * pInfo)

}

(3) OnPrepareDC() 函数

MFC调用OnBeginPrinting() 之后，在打印页面时就为文档中的每一页调用OnPrepareDC()。OnPrepareDC() 程序能够精确地控制MFC把文档的哪一部分输出到打印机中。所以，当应用程序必须打印多页文档时，可重载此函数。还可以使用OnPrepareDC() 来为文档的每一页设置打印机设备描述环境的映射模式和其他属性。

(4) OnPrint() 函数

调用OnPrepareDC() 之后，MFC为每一页调用OnPrint() 。这样，不仅可以在OnPrepareDC() 中设置视区原点 (它控制着文档的哪部分输出到要打印的页上) ，而且还可以在OnPrint() 中进行设置。通常，在把文档的数据以不同方式输出到打印机而不是屏幕上时，可重载OnPrint() 。

当使用AppWizard创建应用程序时，AppWizard不会自动地在视图窗口类中重载OnPrint() 函数。必须使用ClassWizard重载OnPrint() 。

OnPrint() 准确测定文档的哪些行需要在当前页中绘制，并且只绘制这些行。MFC在CPrintInfo对象的成员m_nCurPage中把当前页传递给OnPrint() 。OnPrint() 函数所做的工作主要是：

・ 计算要打印的第一行的行号。

・ 计算要打印的最后一行的行号。

・ 把第一和最后行的行号作为for循环的控制值。

・ 在循环的每次迭代过程中打印一行文本。

(5) OnEndPrinting()

与OnBeginPrinting() 函数相对应的，是OnEndPrinting() 函数。在打印任务完成之后，MFC调用OnEndPrinting() ，使应用程序获得释放OnBeginPrinting() 中分配的资源的机会。AppWizard在应用程序中重载OnEndPrintinp() 函数。

      void CCircleAppView :: OnEndPrinting(CDC *  /*pDC */, cPrintInfo * /* pInfo*/ )

      {

          // 请在此处添加打印之后的清除代码

      }

传递给上述函数的CDC和CPrintInfo对象被注释掉了。如果要访问函数体中的这些对象，需要删除这些注释。
